Etiquetas Obsoletas vs Modernas

Transición de Etiquetas en HTML

Con la llegada de HTML5, muchas etiquetas que mezclaban contenido con presentación fueron declaradas obsoletas. Este cambio responde a la filosofía fundamental de separación de responsabilidades: HTML para estructura, CSS para presentación y JavaScript para comportamiento.

En esta sección analizaremos las etiquetas que han sido deprecadas o eliminadas, y sus equivalentes modernos que ofrecen mejor semántica, accesibilidad y mantenibilidad.

Nuevas Etiquetas Semánticas en HTML5

Estructura Mejorada para Web Accesible

HTML5 no solo eliminó etiquetas obsoletas, sino que introdujo nuevas etiquetas semánticas que mejoran significativamente la estructura y accesibilidad de los documentos web:

Etiquetas Semánticas HTML5
Etiqueta Propósito
<header> Encabezado de página/sección
<nav> Navegación principal
<main> Contenido principal del documento
<article> Artículo o contenido independiente
<section> Sección temática del contenido
<aside> Contenido secundario
<footer> Pie de página o sección final
<figure> Conjunto de contenido con ilustración
<figcaption> Leyenda para <figure>
<mark> Texto resaltado o destacado
<time> Datos temporales con formato estándar

Beneficios de migrar a HTML5 moderno

¿Por qué abandonar las etiquetas obsoletas?

  1. Accesibilidad y SEO: Lectores de pantalla y motores de búsqueda comprenden mejor la estructura semántica del contenido.
  2. Mantenibilidad: Separación clara entre contenido (HTML), presentación (CSS) y comportamiento (JavaScript).
  3. Compatibilidad: Mejor soporte en navegadores modernos y cumplimiento con estándares W3C actuales.
  4. Reutilización: Estilos CSS centralizados aplicables a múltiples elementos y páginas.
  5. Rendimiento: Archivos HTML más ligeros al delegar toda la presentación a hojas de estilo externas.
